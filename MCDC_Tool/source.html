Some notes before using the tool: 25 conditions allowed, the processing time increases as number of conditions increases (e.g. 15 conditions/2 mins, 16 conditions/7 mins)

The input Boolean expression should follow the syntax as described in following examples:

Example 1: ( A | B ) & C | D

Where A, B, C, D represent for independent conditions

Logical operators <AND>, <OR> are entered as single logical symbol <&>, <|>

Example 2: A_1_2 | B | C | A_2 | ( D & A_1 )

Where A_1_2, A_2, A_1 are conditions that depend on each other

A_1_2 and A_2 obey Rule 2 (see Rule description for detail about rules)

A_1_2 and A_1 obey Rule 1

Rule description: there are 5 predefined rules

Rule 1: if 1 condition is FALSE then others must be TRUE

Example:

if( (t_Active_b == FALSE) && ( (t_State_ub != HOLDING_ACTIVE) && (t_State_ub != DRIVE_OFF_STANDBY) && (t_State_ub != DRIVE_OFF_ACTIVE) ) )

Input Boolean expression: A & (B_1 & B_1 & B_1)



Rule 2: if 1 condition is TRUE then others must be FALSE

            Example:

if( (t_State_ub == PARKING_REQUESTED) || (t_State_ub == PARKING_ACTIVE) || (t_State_ub == STAY_PARKING  && t_ActiveSystem_ub == DRIVER_REQUEST) )

Input Boolean expression: A_2 | A_2 | (A_2 & B)



Rule 3: if 1 condition is TRUE then others must be TRUE and vice versa

            Example:

if (((t_State_ub == DRIVE_OFF_STANDBY) && (t_ActiveSystem_ub == DRIVER_REQUESTED)) || ((t_Value_b == FALSE) && (t_ActiveSystem_ub == DRIVER_REQUESTED)))

Input Boolean expression: (A & B_3) | (C & B_3)



Rule 4: if 1 condition is TRUE then others must be FALSE and vice versa

            Example:

if((t_ActiveSystem_ub == ACTIVE_SYSTEM_ACC) || ((t_State_ub == HOLDING_ACTIVE) && (t_ActiveSystem_ub != ACTIVE_SYSTEM_ACC)))

Input Boolean expression: A_4 | (B & A_4)



Rule 5: if "weighted" conditions are TRUE then "non-weighted" conditions are TRUE

            Example:

if(((LongitudinalModeReq_b3 == LONGITUDINAL_MODE_REQ_ENSURE_STANDSTILL) && (ACC_state == ACC_ACC_STATE_ACTIVE)) || ( (ACC_state == ACC_ACC_STATE_DEACTIVATION) && (LongitudinalModeReq_b3 != STOP_MODE_REQ_LONGITUDINAL_GUIDANCE)))

Input Boolean expression: (A_5* & B_2) | (B_2 & A_5)

If the name of a condition has an asterisk *, then the condition is weighted. Otherwise it is non-weighted
